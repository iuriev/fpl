Вопрос 3: Как скомпоновать экран?

Три подхода к структуре GW Review — у каждого свой характер

A

Report Card — вертикальный скролл карточек

Экран читается сверху вниз как «отчёт о неделе». Каждая секция — отдельная карточка. Нарратив: итог → игроки → скамейка → трансферы → вердикт.

GW36 · 68 pts↑ 142k ranks
👥 Игроки (11+4) — sorted by pts
🪑 Скамейка — потеряно 14 pts
🔄 Трансферы — Son→Mbappé +8
✅ Вердикт: трансферы принесли +7

Плюсы

  • Естественный нарратив
  • Просто реализовать
  • Идеально для мобайла

Минусы

  • Много скролла
  • Нет структуры «навигации»
B

Hero + Tabs — шапка фиксирована, контент по вкладкам

Сверху всегда видна шапка с итогами (68 pts, ↑rank). Ниже — две вкладки: «Игроки» и «Решения» (трансферы + скамейка + what-if). Меньше скролла, всё организовано.

68 pts↑ 142kavg 54
👥 Игроки
🔄 Решения
Haaland (C) · 32 pts
Salah · 14 pts
… ещё 9 игроков

Плюсы

  • Итоги всегда на виду
  • Чёткое разделение контента

Минусы

  • «Решения» прячутся во вкладку
  • Чуть сложнее реализация
C

Pitch View + раскрывающиеся панели снизу

Состав на поле с очками каждого игрока (как SquadScreen, но режим «Review»). Под полем — 3 компактных аккордеона: Скамейка / Трансферы / Что-если. Самый визуальный вариант.

⚽ ПОЛЕ (питч-вью с очками)
🟢·32 🟢·14 🟢·12
🟡·8 🟡·7 🔴·2 🔴·1
🪑 Скамейка▾
🔄 Трансферы▾
🤔 Что если…▾

Плюсы

  • Максимально визуально
  • Знакомый питч-лейаут
  • Аккордеоны компактны

Минусы

  • Самый сложный в реализации
  • Питч занимает много места
